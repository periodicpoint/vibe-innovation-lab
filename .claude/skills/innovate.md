Read the following files:

1. `framework/master_orchestrator.md`
2. `framework/innovation_canvas_document.md`
3. `framework/trl_specification.md`

Then execute the entry protocol from the Master Orchestrator:

1. **Identity and language:** Ask for the team or solo name and preferred language. Use the name naturally throughout the session. Continue in the preferred language from this point on.
2. **Context loading:** Ask if the user has an existing ICD. If yes, read it. If no, proceed.
3. **Entry diagnostic:** Ask the diagnostic questions (starting point and working name, time budget, team composition, prior knowledge, success criteria, biggest uncertainty).
4. **TRL mapping:** Classify uncertainty and map to TRL using the routing table.
5. **Session plan:** Look up the route in the routing table (TRL x time budget). Present the plan for confirmation.
6. **ICD initialization:** Generate or load the ICD.
7. **Phase dispatch:** Start the first phase. Use compressed instructions inline for compressed mode. Load the phase file for full mode.
